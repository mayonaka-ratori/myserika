# スキル：返信案生成

## 概要
分類済みの「要返信」メールに対して、Geminiを使って返信案を自動生成するスキル。

## トリガー
- `email-classify.md` スキルで「要返信（重要）」または「要返信（通常）」に分類されたとき

## 手順
1. MEMORY.md からユーザーの返信スタイル・傾向を読み込む
2. contacts.md から送信者情報を取得する
3. Geminiに以下を渡してドラフト生成を依頼：
   - 元メールの内容
   - ユーザーの返信スタイル
   - 送信者との関係
4. 生成されたドラフトを `telegram-notify.md` スキルへ渡す

## プロンプトテンプレート
```
以下のメールに対する返信案を日本語ビジネス文体で生成してください。
[スタイル指示]: {memory_style}
[送信者]: {sender_info}
[元メール]: {original_email}
```

## 出力
- 返信案テキスト（件名・本文）

## 承認フロー
- 承認 → `gmail_client.py` の `send_email()` を呼び出す
- 修正指示 → Geminiに再生成を依頼
- 却下 → STATE.md から削除
